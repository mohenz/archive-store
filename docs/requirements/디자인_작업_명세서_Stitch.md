---

## 1. 프로젝트 개요 및 컨셉

### 1.1. 과업 목적
본 과업은 사용자가 이미지, 텍스트, 문서 파일 등을 개인 저장소에 편리하게 아카이빙하고 관리할 수 있도록 모바일 및 웹 브라우저 환경에 최적화된 반응형 UI/UX 디자인 및 웹 표준 퍼블리싱(HTML/CSS) 산출물을 확보하는 것을 목적으로 합니다.

### 1.2. 디자인 컨셉 및 방향성
- **심플함 및 직관성 (Simple & Intuitive)**: 불필요한 시각적 장식을 배제하고 명료한 그리드 레이아웃을 적용하여, 사용자가 아카이빙된 파일의 종류, 이름, 업로드 시점을 즉각 식별할 수 있는 직관성과 뛰어난 텍스트 가독성을 최우선으로 합니다.
- **최소한의 마이크로 인터랙션**: 파일 드래그 앤 드롭 상태 변화, 업로드 진행률, 클립보드 붙여넣기 성공/실패 여부를 텍스트와 기본 보조 색상 변화만으로 담백하게 안내하는 심플한 UI 피드백을 적용합니다.

---

## 2. 주요 디자인 대상 화면 및 요건

Stitch(Stitch)는 다음 4개 핵심 영역에 대한 UI/UX 디자인과 이를 완벽히 구현한 반응형 마크업 컴포넌트를 제출해야 합니다.

### 2.1. 메인 대시보드 & 파일 탐색기 화면 (반응형)
- **기능 요건**:
  - **스토리지 현황**: 사용 중인 스토리지 용량(예: 1.2GB / 5.0GB)을 가로 막대형 프로그레스 바 및 텍스트 수치로 보여주는 심플한 UI.
  - **카테고리 필터**: 전체, 이미지, 텍스트, 문서 등으로 구성된 플랫한 텍스트 기반의 탭/버튼 스위칭 영역.
  - **파일 리스트/그리드 뷰**:
    - 파일 확장자별 직관적이고 심플한 텍스트/아이콘(예: IMG, TXT, DOC 등 단정한 텍스트 라벨) 및 이미지의 경우 크기가 제한된 플랫 썸네일 노출 영역.
    - 리스트 뷰(목록형)와 카드 그리드 뷰(바둑판형) 간 심플한 레이아웃 전환 버튼.
- **반응형 대응**:
  - **PC (1200px 이상)**: 스토리지 현황 상단 영역 배치 또는 좌측 레이아웃 구성, 파일 그리드 우측 배치.
  - **모바일 (375px~412px)**: 수직 배열 1열 배치로 레이아웃 단순화.

### 2.2. 파일 업로드 제어 UI
- **기능 요건**:
  - **드래그 앤 드롭 영역**: 파일을 화면으로 끌어왔을 때 활성화되는 심플한 오버레이 영역 및 점선 보더 라인 가이드.
  - **다중 업로드 프로그레스 바**: 업로드 중인 파일명과 진행률(%) 및 기본 진행 상태 바(ProgressBar) 노출.
  - **클립보드 업로드 가이드**: 클립보드 붙여넣기(Ctrl+V) 단축키 사용이 가능함을 안내하는 심플한 텍스트 가이드라인 및 간결한 성공/실패 토스트 알림 메시지.

### 2.3. 파일 상세 보기 및 편집 모달(Modal) 창
- **기능 요건**:
  - **이미지 미리보기**: 모달 전체 영역에 이미지를 비율에 맞춰 피팅 렌더링하고, 확대/축소 및 다운로드 버튼 배치.
  - **텍스트 미리보기 및 편집기**: 다운로드 없이 모달 창 내에서 직접 텍스트 내용을 수정하고 저장할 수 있는 인라인 텍스트 에디터(Textarea) 룩앤필 디자인.
  - **문서 요약**: 다운로드하지 않고 PDF 등의 메타데이터를 볼 수 있는 문서 상세 설명 정보 영역.

### 2.4. 사용자 로그인 및 PIN 인증 화면
- **기능 요건**:
  - 텍스트 입력 중심의 심플한 로그인 폼.
  - Single-User용 4자리 또는 6자리 PIN 입력 칸 구성 및 오류 발생 시 붉은색 경고 텍스트 가이드라인 노출.

## 3. 기술 사양 및 납품 요구사항 (Stitch MCP 연동 표준)

Stitch 디자인 부서는 완성된 디자인 및 마크업 코드를 개발팀이 즉각적이고 안정적으로 동기화할 수 있도록, **Stitch MCP(Model Context Protocol) 서버에 등록 및 연동**하여 납품해야 합니다.

1. **디자인 산출물**: Stitch 프로젝트(ID: 3657601612466408108) 내의 각 화면(Design System 등)을 Stitch MCP 서버에 배포 및 연결 처리.
2. **퍼블리싱 산출물**:
   - Stitch MCP를 통해 자동으로 이식 가능한 구조의 HTML5 및 **Vanilla CSS (Pure CSS)** 코드.
   - 반응형 디자인 구현을 위해 Flexbox 및 CSS Grid 레이아웃 설계 필수.
   - 외부 라이브러리 의존성(TailwindCSS 등)은 사용을 배제하며, 순수 CSS만을 활용한 직관적인 UI 마크업 제공.

## 4. 수행 일정 및 임계 경로 통제 (WBS)

Stitch(Stitch)의 전체 작업 기간은 **2주(10 영업일)**로 제한하며, 개발 임계 경로에 맞추어 일정을 통제합니다.

| 일자 | 개발 단계 | 수행 과업 (Task) | 필수 제출물 (Evidence) | 임계 경로 |
| :--- | :--- | :--- | :--- | :---: |
| **D+1 ~ D+3** | 컨셉 정의 | 아카이브 저장소 메인/로그인 화면 와이어프레임 및 스타일 정의 | 스타일 보드, 레이아웃 확정안 | No |
| **D+4 ~ D+6** | 상세 디자인 | 파일 탐색기, 모달 상세 에디터, 업로드 UI 피그마 설계 | 피그마 최종 디자인 시안 | Yes |
| **D+7 ~ D+10** | 마크업 퍼블리싱 | HTML/CSS 반응형 마크업 코딩 및 브라우저 호환성 검증 | HTML/CSS 마크업 코드 (ZIP) | Yes |

## 5. 결론 및 의사결정 유보 시 결과 (Consequence of Inaction)

### 5.1. 의사결정 요청 사항
- **결정 1**: Stitch에 전달할 **CSS 기술 표준(Vanilla CSS)** 최종 승인 여부. (React 컴포넌트 호환성을 극대화하기 위해 순수 CSS 및 변수 활용 구조 권고)
- **결정 2**: 작업 착수일(D+1)을 금일로 확정하고 Stitch의 리소스 투입을 명령할 것인가 여부.

**경영진 결단 유보 시 결과**:
금일 Stitch에 본 작업 명세서가 발송되지 않고 착수가 유보될 경우, 2주차에 예정된 프론트엔드 퍼블리싱 마크업 인수가 순연되어, 3주차 프론트 개발과 4주차 Firebase 통합 개발 전체 일정이 연쇄 지연(Total Schedule Slippage)되는 심각한 일정 노출 상황을 피할 수 없습니다.
