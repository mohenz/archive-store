- **대상 프로젝트**: 개인용 아카이브 저장소 (모바일 및 웹 반응형 서비스)
- **상태**: 기획 및 기능 요건 정의 단계

---

## 1. 개요

본 보고서는 사용자가 제안한 '개인용 아카이브 저장소'의 초기 기능 아이디어를 검토하고, 모바일 및 웹 크로스 플랫폼 환경에서 안정적이고 직관적인 서비스로 작동할 수 있도록 구현 가능성을 검증하며, 실용성을 극대화할 수 있는 보완 및 확대 요건을 정리한 문서입니다.

---

## 2. 제안 기능 기술 검토 및 세부 스펙

사용자가 제시한 6대 핵심 기능 요건에 대한 기술적 실현 가능성 및 세부 구현 규격은 다음과 같습니다.

| NO | 사용자 제안 기능 요건 | 기술 검토 및 세부 구현 규격 | 구현 난이도 |
| :--- | :--- | :--- | :--- |
| **01** | 이미지, 텍스트, 문서 업로드/다운로드 | - 웹 API(File API) 및 multipart/form-data 처리를 통한 파일 전송 구현<br>- 백엔드 스토리지(GCS, S3, 또는 로컬 파일 시스템) 및 메타데이터 DB 연동 필요 | 보통 |
| **02** | 파일 종류별 분류 및 리스트 아이콘 표시 | - 파일 확장자(MIME-Type) 판별 엔진 탑재<br>- 이미지(.png, .jpg), 텍스트(.txt, .log), 문서(.pdf, .docx, .xlsx, .pptx) 유형별 대표 아이콘 매핑 적용 | 쉬움 |
| **03** | 최근 업로드 파일 상단 노출 (내림차순) | - 메타데이터 DB 내 `uploaded_at` (타임스탬프) 컬럼 기준 인덱싱 적용<br>- 최신 업로드 순(`ORDER BY uploaded_at DESC`) 정렬 쿼리 수행 | 쉬움 |
| **04** | 문서 종류별 분류 조회 기능 (필터링) | - 프론트엔드 카테고리 탭(전체, 이미지, 텍스트, 문서 등) 구성<br>- 데이터베이스 쿼리 파라미터 기반 또는 프론트엔드 상태 필터링 처리 | 쉬움 |
| **05** | 이미지 및 텍스트파일 모달 미리보기 | - **이미지**: HTML5 `<img src="objectURL">` 또는 Base64 디코딩을 통한 모달 내 렌더링<br>- **텍스트**: AJAX로 텍스트 스트림 수신 후 HTML `<pre>` 태그 내 안전한 텍스트 렌더링 (XSS 취약점 차단 필수) | 보통 |
| **06** | 직접 파일 선택 및 클립보드 업로드 | - **직접 선택**: HTML `<input type="file" multiple>` 및 드래그 앤 드롭 API 연동<br>- **클립보드**: JavaScript `paste` 이벤트 리스너 등록하여 `event.clipboardData` 내 File 개체(이미지) 또는 Text 데이터 수집 후 업로드 파이프라인 연계 | 보통 |

## 3. 기능 확대 및 보완 제안 (PMO 권장 요건)

사용자 편의성과 서비스 완결성을 높이기 위해 다음과 같은 추가 기능의 도입을 권장합니다.

### 3.1. 업로드 인터페이스 및 편의성 고도화
- **드래그 앤 드롭(Drag & Drop) 업로드**: 파일 선택 창을 열지 않고 화면 전체 또는 특정 영역에 파일을 끌어다 놓아 즉시 업로드할 수 있는 영역 구현.
- **다중 파일 일괄 업로드 및 진행률 표시**: 여러 개의 파일을 동시에 선택해 업로드하고, 각 파일별 업로드 진행 상태(ProgressBar) 및 용량을 퍼센트 단위로 가독성 있게 표시.
- **클립보드 자동 텍스트 파일 변환**: 클립보드에서 텍스트 복사-붙여넣기 시, 임의의 기본 제목(예: `clipboard_20260703_0730.txt`)을 생성하여 자동으로 텍스트 파일 포맷으로 저장 및 아카이빙하는 세부 요건 구현.

### 3.2. 뷰어 및 편집 기능 확장
- **텍스트 파일 실시간 편집 (인라인 메모장)**: 텍스트 파일 미리보기 모달 창 내에 수정 가능 모드(Editable Mode)를 추가하여, 다운로드 없이 웹상에서 텍스트를 즉시 수정하고 저장(업데이트)할 수 있는 기능.
- **PDF 및 오피스 문서 미리보기 지원**: Google Docs Viewer API 또는 pdf.js 라이브러리를 연동하여, 다운로드 없이 브라우저 내에서 직접 PDF 및 주요 문서 요약본을 미리 볼 수 있는 인라인 뷰어 지원.

### 3.3. 검색 및 분류 체계 강화
- **키워드 및 메타데이터 검색**: 파일명, 확장자, 업로드 기간 기준의 다중 조건 검색 바(Search Bar) 구현.
- **사용자 커스텀 태그(Tag) 관리**: 파일별로 해시태그(예: #업무, #개인, #영수증)를 추가할 수 있게 하고, 태그 클릭 시 해당 태그가 달린 파일들만 그룹화하여 조회할 수 있는 논리적 분류 체계 추가.

### 3.4. 스토리지 자원 관리
- **스토리지 사용 현황 대시보드**: 개인용 저장소의 전체 할당 용량 대비 현재 사용 용량(예: 1.2GB / 5.0GB)을 차트 형태로 시각화하여 제공.
- **중복 파일 사전 경고**: 동일한 파일명이나 해시값(MD5/SHA-256)을 가진 파일이 중복 업로드될 경우, 덮어쓰기 또는 새 이름으로 저장할지 선택할 수 있는 경고 시스템.

## 4. 비기능 및 인프라/보안 요구사항

본 서비스가 개인용 민감 자산을 다루는 저장소임을 감안하여 아래의 비기능 가이드라인이 준수되어야 합니다.

### 4.1. 보안 및 권한 요건 (Security)
- **HTTPS 보안 암호화 통신**: 파일 업로드 및 다운로드 시 제3자의 데이터 가로채기를 방지하기 위해 SSL/TLS 암호화 적용 필수.
- **사용자 인증 및 세션 제어**: 개인 저장소 접근 제한을 위해 간소화된 PIN 번호 로그인 또는 소셜 로그인(OAuth) 연동 필요.
- **접근 권한 격리**: 멀티 유저 확장성을 염두에 두고 사용자 ID별 물리적 저장 경로 격리 처리 (`/storage/{user_id}/files/`).

### 4.2. 성능 및 반응형 레이아웃 (Performance & Layout)
- **모바일 웹 반응형 그리드**: CSS Grid/Flexbox 및 뷰포트 미디어 쿼리를 설계하여, 모바일 환경에서는 한 열로 목록이 배치되고 웹(PC) 환경에서는 카드 그리드 형태로 자동 전환되는 룩앤필 제공.
- **이미지 썸네일 자동 생성 및 레이지 로딩**: 대용량 이미지 업로드 시 리스트 뷰 속도 유지를 위해 썸네일 이미지를 백엔드에서 자동 리사이징 생성하고, 화면 스크롤에 맞춰 썸네일을 로드하는 레이지 로딩(Lazy Loading) 기법 적용.
